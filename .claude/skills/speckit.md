# Spec Kit - 仕様駆動開発ワークフロー

このスキルは、機能開発を段階的に進めるための統合ワークフローを提供します。

## ワークフローの概要

```
1. 仕様定義 → 2. 計画策定 → 3. タスク分解 → 4. 実装 → 5. レビュー
```

## 実行フロー

### Phase 1: 仕様定義

ユーザーから機能の説明を受け取り、`spec.md` を作成します。

**実行コマンド**: `/speckit.specify "機能の説明"`

**出力**:
- ブランチ作成（例: `1-user-auth`）
- `specs/1-user-auth/spec.md` - 機能仕様書
- `specs/1-user-auth/checklists/requirements.md` - 品質チェックリスト

**このフェーズでやること**:
- ユーザーニーズの明確化
- 機能要件の定義
- 成功基準の設定
- 曖昧さの排除

---

### Phase 2: 技術計画

仕様をもとに技術的な実装計画を策定します。

**実行コマンド**: `/speckit.plan`

**出力**:
- `plan.md` - 技術スタック、アーキテクチャ、構成
- `research.md` - 技術調査結果
- `data-model.md` - データモデル設計
- `contracts/` - API契約
- `quickstart.md` - テストシナリオ

**このフェーズでやること**:
- 技術スタックの選定
- アーキテクチャ設計
- データモデル定義
- API設計

---

### Phase 3: タスク分解

計画を実行可能なタスクに分解します。

**実行コマンド**: `/speckit.tasks`

**出力**:
- `tasks.md` - 依存関係順のタスクリスト
- 並列実行可能なタスクの特定
- 各ユーザーストーリーの独立テスト基準

**このフェーズでやること**:
- 実装タスクの洗い出し
- 依存関係の整理
- 実行順序の最適化
- MVP範囲の特定

---

### Phase 4: 実装

タスクを順次実行していきます。

**実行コマンド**: `/speckit.implement`

**このフェーズでやること**:
- タスクの段階的実装
- テスト作成（要求がある場合）
- コードレビュー
- コミット

---

### Phase 5: 品質チェック（オプション）

**実行コマンド**: `/speckit.analyze`

**このフェーズでやること**:
- 仕様・計画・タスクの一貫性チェック
- 品質問題の特定

---

## 使い方の例

### 例1: 新機能の完全実装

```
User: "ユーザー認証機能を追加したい"

# Speckitスキルが起動

Step 1: /speckit.specify "ユーザー認証機能を追加したい"
→ specs/1-user-auth/spec.md が作成される

Step 2: /speckit.plan
→ 技術計画とデータモデルが作成される

Step 3: /speckit.tasks
→ 実装タスクリストが作成される

Step 4: /speckit.implement
→ タスクを順次実装

Step 5: /speckit.analyze（必要に応じて）
→ 品質チェック
```

### 例2: 既存の仕様から実装

```
User: "既にspec.mdがあるので、計画から始めたい"

# Speckitスキルが起動

Step 1: /speckit.plan
→ 既存のspec.mdを読み込んで計画策定

Step 2以降は同じ
```

## よくある質問

**Q: スラッシュコマンドを直接実行するのとの違いは？**
A: スキルを使うと、ワークフロー全体のガイダンスが得られます。各ステップの目的や次のアクションが明確になります。

**Q: 途中のステップから始められる？**
A: はい。既にspec.mdがあれば `/speckit.plan` から、plan.mdがあれば `/speckit.tasks` から開始できます。

**Q: 自動で全部やってくれる？**
A: このスキルは手動ガイド用です。完全自動化が必要な場合は、workflow-coordinatorエージェントを使用してください。

## 次のステップ

1. まず `/speckit.specify "機能の説明"` で仕様を作成
2. 生成された仕様を確認・調整
3. `/speckit.plan` で技術計画を策定
4. 以降、段階的に進める

各ステップで生成されたファイルは、必要に応じて手動で編集できます。
